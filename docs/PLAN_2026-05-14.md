## Priority 2 — Productionisation Polish

### P2.1 — Migration 022 (drop legacy IG columns)
Create `supabase/migrations/022_drop_legacy_instagram_columns.sql` (skipped during this build, intentionally separated):
```sql
ALTER TABLE influencer_profiles
  DROP COLUMN IF EXISTS instagram_access_token,
  DROP COLUMN IF EXISTS instagram_token_expires_at,
  DROP COLUMN IF EXISTS instagram_ig_user_id;
```
**Apply only after** main has been live for ~1 week and `grep -r instagram_access_token server/src` returns nothing.

### P2.2 — `socialStats.ts` fallback path
Currently `/api/social/stats/:userId` reads from `creator_platform_metrics`. For creators with no OAuth connection, it falls back to mock. After migration 021 is live, audit which creators have neither IG nor YT OAuth (most of the 9 seeded creators don't) and decide UX:
- Keep mock fallback (current behaviour, low effort)
- Show "Not connected" placeholder instead of synthetic data (cleaner UX, higher trust)

### P2.3 — `socialStats.ts` engagement rate display fix
Subtle bug: new code stores `engagement_rate` as percent number (e.g. `4.2` meaning 4.2%). Mock path returns it as fraction (e.g. `0.042`). Frontend treats both inconsistently. Pick one and normalise everywhere. Recommend: store + return as percent number; multiply by 100 only when reading from `creator_platform_metrics`.

### P2.4 — Initial sync race
`syncInstagram` / `syncYouTube` are fired with `.catch()` after callback redirect. If they're slow (~3-5s for IG insights), the user lands on dashboard before metrics populate. UI currently shows "No data yet — click Refresh". Either:
- Make user wait on a loading page (server-rendered intermediate) — heavier
- Add automatic refetch every 5s for first 30s after `?ig_connected=true` in URL — easier, do this

### P2.5 — Token rotation notification
For Instagram (60-day page token): when `token_expires_at - now() < 7 days`, send email/notification to creator to reconnect. Cron path: extend `syncAllPlatforms` to detect + queue notification. Saves silent failures.
